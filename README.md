## **OOP erklärt:**

* __Abstraktion__
  
  Jedes Objekt im System kann als Modell eines Akteurs betrachtet werden, der Aufträge erledigen, seinen Zustand berichten und ändern und mit den anderen Objekten im System kommunizieren kann, ohne offenlegen zu müssen, wie diese Fähigkeiten implementiert sind.
* __Kapselung__

  Auf die interne Datenstruktur kann nicht direkt zugegriffen werden, sondern nur über definierte Schnittstellen. Objekte können den internen Zustand anderer Objekte nicht in unerwarteter Weise lesen oder ändern. Ein Objekt hat eine Schnittstelle, die darüber bestimmt, auf welche Weise mit dem Objekt interagiert werden kann.
* __Polymorphie__

  Fähigkeit eines Bezeichners, abhängig von seiner Verwendung unterschiedliche Datentypen anzunehmen. Verschiedene Objekte können auf die gleiche Nachricht unterschiedlich reagieren. Wird die Art der Reaktion auf die Nachricht erst zur Laufzeit aufgelöst, wird dies auch späte Bindung genannt.
* __Vererbung__

  abgeleitete Klasse besitzen die Methoden und Attribute der Basisklasse. Neue Arten von Objekten können auf der Basis bereits vorhandener Objektdefinitionen festgelegt werden. Es können neue Bestandteile hinzugenommen werden oder vorhandene überlagert werden.


## **Design Patterns erklärt:**
Design Patterns sind Vorlagen, bzw. Lösungsschablonen, die oft vorkommende Programmierprobleme lösen. Man kann auch ohne Design Patterns diese Probleme umgehen, allerdings zählt das Anwenden von Design Patterns als beste, oder zumindestens weitgehend akzeptierte, Lösung.
